Grid & Layout

Documentation on different layout and grid helpers

Container

Container is the basic devkit layout element. By default it will apply the bundle max-width and side gutters. Be wary that the container element will not apply the flexbox grid by default. Container can be used in legacy browsers for a fluid container.

this is a container element 

<div class="container grid">
	...
</div>

Grid

The grid class turns any element into an equal parts divisible flexbox grid. If no column number is specified with the .col class it will automatically set the grid columns for you. This is usefull if you have a server rendered app that will render different amount of results and you wish to adapt your UI with it.

Columns

Columns is the best way to play with the grid. Any container with the .grid class can be split by 12 sections, any of those 12 childrens can also be split again. You can change the number of columns you want with the $grid-columns variable. If you go over 12 columns for a given row, this column is automatically going to be sent to the next row.

Form grid

You can use all the grid layouts class to help you build more complex forms. You can also use .form-col helper to automatically give equal spacing between all your form elements. If you wish to have equal spacing vertically as well you can wrap your feilds inside a .form-group element. By default Devkit does not provide media queries for simplifying forms for mobile. One quick trip is to simply set all the .form-col to 100% on mobile, then all the fields will be one below the others.

Form grid utilities

Here is some common UI paterns fix that you will find handy.

To line up vertically labels againts inputs you can use the .form-inline-label helper to get the job done. Simply use it on a .form-col element.

To line up two fields in different colums and one does not have a label use .form-empty-label on an empty label element. If you want the two elements to merge on top of eachoter on mobile simply target this element and add a display: none.

To group form inputs together you can use the .form-input-group helper class. Basically anything that has borders can be merged into groups.

Iconed inputs

One day you may feel the need to do some extra work and add icons to your input fields... Fear not, we did all the work for you, simply put a span around your icon, then put a wrapper that contains the input and the span with the .form-icon-input class. you can also add the .left or .right modifier to align your icon to your liking, it will smartly position it were it needs to be !

Form perfect checkboxes

Pesky browsers are not all the same, and sometimes your deligthfully coded form will look like smelly porridge. To prevent this, you can use these hacky methods to have the same inputs on mostly any browsers.


  • required dkit-checkbox.js
  • important We reccomend that you only set the checked status using the .checked class on the .fix-checkbox element, the script will take care of automatically setting the invisible input to the correct state on page load. If you need to add this field asyncronously, you can render the element with the correct class and the correct "checked" attribute on the input yourself.
  • To make it work simply use the [data-checkbox-name] attribute on the .fix-checkbox element. Make sure that the .fix-checkbox-group element contains a checkbox or a radio input.
  • Using the class .radio on the .fix-checkbox element with change the display and the behavior of the checkbox to match a regular checkbox.
